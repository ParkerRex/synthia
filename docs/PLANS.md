# Plans

This file routes long-running implementation work in synthia.

Use `docs/programs/` for durable multi-slice initiatives that sit above several ExecPlans. Use `docs/exec-plans/` for one bounded implementation slice with its own validation loop.

## Routing

- Product requirements belong in `SPEC.md`.
- Project vocabulary belongs in `CONTEXT.md`.
- Initiative-level state belongs in `docs/programs/active/<YYYY-MM-DD-program-id>/program.md`.
- Slice-level execution instructions belong in `docs/exec-plans/active/YYYY-MM-DD-<slug>.md`.
